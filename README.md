Criar um jogo do gênero de suspense com elementos do folclore brasileiro utilizando a sintaxe do Portugol.

## 🎮 Como jogar

1. Baixe o arquivo `O_Ultimo_Desafio_Da_Bruxa.por` na sua máquina.
2. Acesse no seu navegador a plataforma [portugol.dev](https://portugol.dev/).
3. Na página inicial do Portugol Web Studio, clique em **Abrir Arquivo** (ou use o atalho `Ctrl + O`).
4. Selecione o arquivo `.por` que você acabou de baixar.
5. Pressione o botão **Play** (ícone de triângulo ▶️) localizado na parte superior esquerda para executar o código.

## 🎮 Estrutura Lógica

Estruturas de Dados: Uso de vetores para persistência de dados em tempo de execução (como historicoJogador e historicoPalitos).
Modularização: Divisão limpa do fluxo do software em funções específicas para isolar responsabilidades (telas, lógica de jogo, cutscenes).
Controle de Fluxo Complexo: Uso de condicionais aninhadas (se/senao), laços de repetição (enquanto, para, faca-enquanto) e estruturas de escolha (escolha/caso) para controle de estados.
Manipulação de Tipos e Bibliotecas: Uso prático de bibliotecas nativas (Util para temporização e Texto para tratamento de strings).


## Regras e requisitos técnicos:
Narrativa interativa (texto):
Criar uma história dividida em "cenas", "capítulos" ou "fases", apresentando escolhas para o jogador a cada etapa.
A história deve envolver elementos de suspense, mistério ou terror baseados em lendas urbanas.

## Capa de entrada (obrigatório):
O jogo deve iniciar com uma tela de apresentação animada.
A capa deve conter o nome do jogo e ambientação temática.
É fortemente recomendado o uso de ASCII Art (imagens feitas com caracteres) para representar o título ou cenário.

## Animações durante o jogo (obrigatório):
O jogo deve conter efeitos animados em momentos importantes da história.
Exemplos:


Texto aparecendo aos poucos (efeito de digitação)
Pontos de suspense (...)
Simulação de movimento usando repetição

Uso de imagens em ASCII (diferencial criativo):
Inserir imagens em ASCII (ASC) para representar:


Cenários (ex: casa assombrada, floresta, rua escura)
Personagens ou criaturas
Momentos importantes da história
As imagens podem ser:


Criadas manualmente
Convertidas de imagens reais para ASCII usando ferramentas online
Exemplo simples de ASCII:
   /\\
  /  \\      CASA ABANDONADA
/____\\
| [] |
|    |

## Decisões e caminhos diferentes:
Cada escolha leva a outra parte da história.
Pelo menos 3 finais diferentes:


Vitória
Derrota
Final alternativo

## Funções:
Criar funções para cada cena ou decisão relevante da história.
Ex: inicio(), casaAssombrada(), encontroCriatura(), finalVitoria() etc.

Vetores e/ou matrizes:
Usar um vetor para registrar o histórico de decisões do jogador.
Opcional: matriz para mapear caminhos da história.

Estruturas de repetição e decisão:
O jogo deve manter um loop até chegar a um final.
Usar enquanto, para, se, senao.

Sistema de pontos (opcional):
Pontos por decisões inteligentes ou descobertas.

## Entregáveis (07/04):
Código completo comentado.
História documentada.
Uso de ASCII e animações será considerado diferencial criativo.
Apresentação do jogo.

Sugestões de tema:
Casa abandonada assombrada
Estrada com aparição misteriosa
Escola com eventos paranormais
Criatura urbana desconhecida
Mistério envolvendo desaparecimentos

## Objetivos do Desafio:


Desenvolver lógica de programação com estruturas de decisão e repetição.


Aplicar programação em narrativas interativas.


Organizar código de forma clara.


Estimular criatividade com recursos visuais em texto (ASCII).
